SQL Serverの実行プランを、URLひとつで共有。
パフォーマンス改善の議論を、速く・正確に進めるためのツールです。


https://queryplansharev5.vercel.app/


### こんなときに効きます

- クエリ改善レビューするとき
- チャットで実行プランを貼っても「見えない…」となったとき
- 過去に共有した計画を探して掘り返すとき

## 作った背景（現場の困りごと → 解決の設計判断）


クエリ改善の相談をするとき、実行プランをChatなどに貼ると **整形が崩れて読めない** **詳細が把握できない**、結局「どの環境？どのSQL？どのプラン？」がスレッド内に散らばってしまいます。


その結果、レビュー自体よりも前提のすり合わせに時間が溶けがちでした。


そこで、**テキストの実行プランを貼るだけで、見やすいURLにして共有**できるようにしました。


### こだわった点

- **ツリー表示**: ボトルネック候補を追いやすくする
- **テーブル表示**: 確認すべき内容を見えるように、コピペも可能！
- **アンリステッド共有**: URLを貼るだけでレビューを始められる
    - 共有URLの扱いと、必要に応じた閲覧範囲の制御（限定公開）を両立動き
- **ダッシュボード**: 共有したプランを後から見つけられるようにする（ログイン時）

## QueryPlanShareでできること


### 貼るだけ共有（URL発行）

- クエリプランを貼り付けて**アンリステッド**リンクを発行
- Slack / GitHub Issue / PRにURLを貼るだけ

### ツリーで読みやすく（ボトルネック追跡）

- テキストのプランを、読みやすいツリー表示へ
- ボトルネック箇所を追いやすい

### 共有したプランを一覧管理

- 共有したプランをダッシュボードで一覧管理
- 編集・削除など、登録者がコントロール（ログイン時）

---


## 使い方

1. SQL Serverのクエリプラン（テキスト）を貼り付け
2. **アンリステッド**リンクを生成
3. チームにURL共有してレビュー開始

---


## 限定公開（閲覧範囲の制御）


アンリステッドリンクを知っている人が閲覧できる前提です。必要に応じて「限定公開」で閲覧範囲を制御できます（ゲスト / ログインどちらでも運用可能）。

## FAQ


### Q. どのDBの実行プランに対応していますか？


A. 現状は SQL Server のクエリプランを対象にしています。


### Q. 共有したデータは公開されますか？


A. 「限定公開」にすることで閲覧範囲を制御できます（ゲスト / ログインどちらでも運用可能）。アンリステッドリンクを知っている人は閲覧できる形になります。


### Q. サインアップ／ログインのメリットは？


A. ログイン後に登録したプランは、登録者が登録したものの一覧可能で、登録者が編集・削除などをコントロールできます。
